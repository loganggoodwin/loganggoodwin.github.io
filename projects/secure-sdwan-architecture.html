🔒 Secure SD‑WAN Network Architecture

Designed a future‑state enterprise network for a rapidly growing multi‑site organization using a dual‑hub SD‑WAN topology with consistent security controls, high availability, and centralized visibility.

Download PDF • Contact

TL;DR

  • Problem: Inconsistent site security + WAN single points of failure + scaling pressure.
  • Solution: Dual‑hub SD‑WAN with redundant ISPs, NGFW everywhere, Zero Trust access, segmentation, and centralized monitoring.
  • Outcome: A scalable design that improves uptime, standardizes security, and supports voice/video/cloud workloads.

Context & Constraints

  • Rapid growth required a network that scales cleanly without redesigning every time a site is added.
  • Reliability mattered: business communications (VoIP/video/data sync) must remain stable during link issues.
  • Security had to be consistent across all locations (closing gaps where protections differed by site).

Architecture (Future State)

Dual‑Hub SD‑WAN Topology

  • Primary hub: Dallas
  • Secondary hub / DR: Houston
  • Spokes: Branch sites connected via encrypted SD‑WAN tunnels
  • Resilience: Dual ISP circuits at hubs + automatic failover and dynamic path selection

Security Stack

  • NGFW at every site (consistent inspection, IPS, application control)
  • Zero Trust principles (strong identity checks and reduced implicit trust)
  • MFA for privileged access and remote access paths
  • Centralized monitoring (NetFlow/telemetry + SIEM visibility)

Segmentation & Access Control

  • Standard VLANs across sites (data / voice / video / servers / management / guest)
  • ACLs to limit east‑west traffic and reduce attack surface
  • 802.1X / NAC for device validation before access

Performance & QoS

  • QoS policies to protect voice/video from latency, jitter, and packet loss
  • Application‑aware routing to keep critical traffic on the best path

Why These Choices

  • SD‑WAN: Simplifies multi‑site connectivity while improving resiliency and application performance.
  • Dual hubs + multi‑ISP: Reduces outage risk and supports disaster recovery continuity.
  • NGFW everywhere: Eliminates inconsistent protections between sites.
  • Segmentation: Limits blast radius if a user/device is compromised.
  • Centralized visibility: Faster detection/response with consistent telemetry.

Skills Demonstrated

  • Enterprise network architecture design (WAN + campus)
  • SD‑WAN topology and resiliency planning
  • Network security controls (NGFW, segmentation, NAC)
  • Zero Trust access concepts + MFA strategy
  • Monitoring/telemetry approach (SIEM + flow visibility)

What I’d Improve Next

  • Define SIEM detections for SD‑WAN/NGFW telemetry and baseline anomalies.
  • Run failover / DR simulations with measurable RTO/RPO targets.
  • Automate configuration compliance checks (golden templates + drift detection).
