Overview

I think strong policy turns security expectations into repeatable operations. This project focuses on enforceable, real-world policy language: scope, accountability, and user behavior controls.

What This Covers

  • Policy purpose, scope, roles/responsibilities, and enforcement
  • Acceptable use + email/internet guidance and social media boundaries
  • Remote access expectations (authentication, approved tools, monitoring)
  • Access control principles (least privilege, reviews, exceptions)
  • Incident reporting and escalation expectations

Skills Demonstrated

  • Security governance and policy development
  • Risk-to-control translation
  • Clear documentation for technical and non-technical stakeholders

What I Would Improve Next

  • Map policy sections to NIST 800-53 control families
  • Add an exception workflow + annual review cadence
  • Create a one-page policy index for fast navigation
