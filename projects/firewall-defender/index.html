How Firewalls Work

A firewall is a security control that filters traffic between networks. It checks protocol, source, destination, port numbers, connection state, and policy rules.

1. Inspect the packet

The game shows source, destination, protocol, source port, destination port, service, and connection state. The hint feature can walk players through these fields when they get stuck.

2. Match a rule

Rules are checked from top to bottom. The first matching rule usually decides whether traffic is allowed or blocked.

3. Apply least privilege

Allow only traffic the network actually needs. Management ports, databases, and file sharing should be tightly restricted.

4. Use default deny

A secure posture blocks traffic unless there is a clear business reason and a specific rule to allow it.

5. Segment the network

Database, server, user, internet, and management zones should not all be treated the same.

6. Think statefully

Stateful firewalls can allow return traffic for sessions that were started by trusted systems while blocking unsolicited inbound traffic.

Teaching goal: Firewall Defender demonstrates packet inspection, firewall rule order, ports, protocols, least privilege, segmentation, default deny, and stateful filtering.
